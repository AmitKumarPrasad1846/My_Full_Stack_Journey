Essential concepts for JavaScript developers

Variables & Declaration

let Declaration

let a = 5; // Block-scoped

Block-scoped variable that can be reassigned.

const Declaration

const PI = 3.14; // PI = 3.15 → Error

Block-scoped constant that cannot be reassigned.

Variable Naming Rules:

  • Case-sensitive (a ≠ A)
  • Cannot start with numbers (55a is invalid)
  • Can include _ or $ (e.g., _a)

Data Types (7 Primitive Types)

Type Example Description
Number let age = 23; Integer or floating-point numbers
String let name = "Amit"; Textual data
Boolean const isStudent = true; Logical true/false
Undefined let x; Default value of uninitialized variables
Null let y = null; Intentional empty value
Symbol const id = Symbol(); Unique identifiers
BigInt const bigNum = 123n; Large integers

Checking Types

console.log(typeof age); // "number"

Objects

let person = { name: "Harry", // Simple key "job code": 5600, // Multi-word keys need quotes is_handsome: true // Boolean value };

Dynamic Properties

Objects can be modified after creation:

person.salary = "100crores"; // Add new property
person.salary = "500crores"; // Update property

Scope

{ let a = 66; // Only exists inside this block } console.log(a); // Error (if 'a' was defined outside)

var is function-scoped (avoid in modern JS)

Always prefer let and const over var.

Best Practices

Variable Declaration

  • Use const by default
  • Only use let if reassignment is needed

Naming Conventions

  • Use camelCase (userName)
  • Multi-word object keys need quotes ("job code")

Tooling

  • Use linting tools to catch errors
  • Watch for duplicate variable names

Key Takeaways

  • JavaScript is case-sensitive
  • Prefer let/const over var
  • Objects are mutable (properties can be added/changed dynamically)
  • 7 primitive data types in JavaScript
