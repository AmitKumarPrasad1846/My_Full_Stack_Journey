Introduction to CSS Animations

CSS animations allow you to create smooth, visually appealing transitions and movements on your web pages without JavaScript. They're performant, easy to implement, and can significantly enhance user experience when used properly.

When to Use Animations

Purpose Example
Drawing attention Pulsing call-to-action buttons
Visual feedback Button press effects
State changes Showing/hiding elements
Page transitions Smooth route changes
Loading indicators Spinners, progress bars

Transition vs Animation

Transition Animation
Triggered by events (hover, focus etc.) Can run automatically or on page load
Only animates between two states Can have multiple @keyframes (multi-step)
Simpler implementation More powerful and flexible
Good for simple interactions Ideal for complex sequences
/* Transition example */
.button {
  transition: all 0.3s ease;
}

/* Animation example */
.box {
  animation: slideIn 1s ease 0.5s both;
}
@keyframes slideIn {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}

Animation Properties

Property Description Example Values
animation-name Specifies the name of the @keyframes rule slideIn, fadeOut, bounce
animation-duration How long one cycle should take 2s, 500ms, 0.5s
animation-timing-function Speed curve of the animation ease, linear, ease-in-out
animation-delay Delay before animation starts 1s, 200ms, 0s
animation-iteration-count How many times to play 1, infinite, 3
animation-direction Normal, reverse, or alternate normal, reverse, alternate
animation-fill-mode Styles before/after animation none, forwards, backwards
animation-play-state Pause or run the animation running, paused

Shorthand Syntax

.element {
  animation: name duration timing-function delay iteration-count direction fill-mode play-state;
}

/* Practical example */
.box {
  animation: pulse 2s ease-in-out 0.5s infinite alternate;
}

@keyframes Rule

The @keyframes rule is where you define what the animation should do at various points during its sequence.

@keyframes animationName {
  0% { property: value; }
  50% { property: value; }
  100% { property: value; }
}

/* From/To syntax (equivalent to 0% and 100%) */
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

Keyframe Percentage Tips

  • You can have as many percentage points as needed (5%, 30%, 77%, etc.)
  • Percentages don't need to be in order - the browser will sort them
  • If multiple keyframes share the same properties, you can group them: 0%, 20%, 100% { opacity: 1; }
  • You can animate multiple properties simultaneously

Practical Animation Examples

Bounce Effect

Bounce!
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-30px); }
}

.element {
  animation: bounce 2s infinite;
}

Fade In

Fade In
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

.element {
  animation: fadeIn 3s ease-in;
}

Continuous Rotation

Rotate
@keyframes rotate {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

f .element {
  animation: rotate 4s linear infinite;
}

Animation Performance

For smooth animations that don't slow down your page, follow these guidelines:

Do Don't
Use transform and opacity Animate width, height, or margin
Use will-change for complex animations Animate too many elements simultaneously
Keep animations short (under 300ms for UI) Create long, complex animations
Use requestAnimationFrame for JS animations Use setInterval for animations

Best Properties to Animate

These properties are most efficient for animations:

  • transform (translate, scale, rotate)
  • opacity
  • filter (with caution)
  • clip-path (modern browsers)

Advanced Techniques

Chaining Animations

Use animation-delay to sequence multiple animations:

.box1 { animation: fadeIn 1s forwards; }
.box2 { animation: fadeIn 1s 1s forwards; }
.box3 { animation: fadeIn 1s 2s forwards; }

Animation Events in JavaScript

You can detect when animations start, end, or repeat:

const element = document.querySelector('.animated');

element.addEventListener('animationstart', () => {
  console.log('Animation started');
});

element.addEventListener('animationend', () => {
  console.log('Animation finished');
});
