What are CSS Transitions?

CSS transitions allow you to create smooth, gradual animations between different states of an element. They provide control over how property changes occur over time, making your interfaces feel more dynamic and responsive without JavaScript.

transition: property duration timing-function delay | none | initial | inherit;

Transition Properties

Property Purpose Example Values
property Specifies which CSS property to animate (use "all" for multiple properties) background-color, transform, opacity, all
duration How long the animation takes to complete 0.3s, 1s, 200ms
timing-function Controls the acceleration curve of the animation (how the animation progresses over time) ease, linear, ease-in-out, cubic-bezier()
delay How long to wait before starting the animation 0s, 0.5s, 1s

Best Practices

  • Use transitions for simple state changes (hover, focus, active, class changes)
  • Prefer transform and opacity for best performance (they don't trigger layout/paint)
  • Keep transition durations between 200ms-500ms for UI elements
  • Use consistent timing across your application for cohesive feel
  • Avoid transitioning layout properties like width/height (use transform: scale instead)
  • Consider using will-change for elements you know will be animated
  • Test animations on lower-powered devices to ensure performance
